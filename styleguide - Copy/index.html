Design principles

We add values, not features

All features on the user interface must add value to the system and the user.

We are consistent in the way we design

We are consistent in pur design, language, and coding. We believe that a consistent design is a better design. We make use of well-known UI patterns, recognized icons, easy to understand language, and uniform terminology.

We design based on data

We love data and make decisions based on it.

We design simple and intuitive user interfaces

We design user interfaces that work as the users expect and are easy to learn. We use a logical information hierarchy and create a visual hierarchy that is easy to scan and easy to understand so the users easily og quickly can perform the tasks they need.

We give users control

We design user interfaces that make users feel they are in control. We do this by making our system predictable, warn of errors and actions, and make it easy to cancel unwanted actions. We give the users feedback they understand. We show them errors and guide them to fix them.

We design solutions for all screen sizes

We design user interfaces that have the same interface regardless of screen size. It is designed to scale optimally and fit all screen sizes.
